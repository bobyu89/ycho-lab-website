## 1. 目標

以使用者提供的粉色護理研究室範例圖為視覺基準，建置多頁式實驗室網站，內容架構依「五層模型」完整展開：

| 層級 | 功能 | 對應 |
|---|---|---|
| 品牌層 | 讓人記住研究室定位 | 首頁 Hero、標語、使命 |
| 學術層 | 建立可信度 | 研究主題、論文、計畫 |
| 教育層 | 呈現護理教育價值 | 教學資源頁（佔位） |
| 合作層 | 吸引跨域合作 | 合作交流頁（佔位＋照片牆） |
| 資源層 | 網站長期價值 | 開放資源頁（佔位） |

### 非目標（這版不做）
- 中英全站切換
- 後台管理介面／會員系統
- 部落格／文章系統
- 由 Claude 手繪 SVG 人物插畫（使用者明確否決）

## 2. 技術方案（沿用已確認決策）

- **純靜態網站**：HTML + CSS + 原生 JS，無框架、無建置工具，Firebase Hosting 部署
- **共用頁首頁尾**：`js/components.js` 模板注入，各頁引用
- **最新消息**：讀公開發布的 Google 試算表（gviz JSON 端點）
- **活動照片**：讀「知道連結者可檢視」的 Google 雲端資料夾（Drive API v3 + API 金鑰，縮圖 `drive.google.com/thumbnail?id=...&sz=w800`）
- **降級策略**：任一外部資料源失敗時顯示內建靜態備用內容，版面不破
- **聯絡表單**：mailto 方式——填寫後點送出，組出主旨與內文開啟使用者的郵件軟體寄至老師信箱（已確認）

## 3. 視覺系統（依範例圖）

### 配色

| 角色 | 色票 | 用途 |
|---|---|---|
| 主色 | #E85D8A 玫瑰粉 | 標題強調、圖示、裝飾 |
| 主色深 | #D6336C 深粉 | 內文連結、需要對比的粉色文字 |
| 輔色 | #F59B53 珊瑚橘 | CTA「加入我們」按鈕、次要強調 |
| 底色 | #FFFFFF 白 | 頁面基底 |
| 區塊底 | #FDEEF2 淺粉 | 交錯區塊背景、Hero 背景基調 |
| 卡片變化 | #FFF6E8 奶油黃 | 部分核心領域卡片底 |
| 文字 | #3D3A3A 暖炭灰 | 標題、內文 |

無障礙原則：文字一律用 #3D3A3A 或 #D6336C 深粉，達 WCAG AA；淺粉/珊瑚橘僅作背景、按鈕底（配白字需驗對比）與裝飾。

### 造型語言
- 大圓角卡片（16–24px）、膠囊按鈕、粉色柔和陰影（`rgba(230,93,138,0.15)` 級）
- 愛心＋十字 logo、點陣 pattern、心電圖裝飾線、柔和 blob
- 字體：Noto Sans TC（400/500/700/900）；英文小標可用 Inter

### 圖片素材策略（已確認：不由 Claude 手繪人物）
- 基底設計不依賴人物插畫：以漸層、圖示（inline SVG icon 等級可以）、心電圖線撐起風格
- Hero 與各區大圖＝規格化 `<img>` 插槽 + 粉色漸層佔位圖
- 交付 `assets/素材規格清單.md`：每個插槽的尺寸、比例、建議 AI 生圖 prompt；使用者換檔即完成替換
- 團隊成員照片：先用姓名縮寫圓形頭像佔位

### 動畫（溫和取向）
| 位置 | 動畫 |
|---|---|
| 全區塊 | 捲動進場淡入上滑，卡片群錯開 0.1s（IntersectionObserver） |
| 卡片/按鈕 | hover 浮起、微放大 |
| 統計數字 | 進場 0 滾動到目標值 |
| 心電圖線 | 緩慢橫向流動 |
| 照片牆 | hover 微放大＋說明條、燈箱 |

全部包在 `prefers-reduced-motion` 檢查內。

## 5. 首頁版面（依範例圖結構）

1. **頁首導覽**——白底、logo（愛心十字＋中文名＋英文小字）、導覽、珊瑚橘「加入我們」CTA
2. **Hero**——淺粉背景＋裝飾（點陣、心電圖線、blob）；左：大標語（粉色關鍵字強調）＋副文案＋雙按鈕（探索研究／查看成果）＋四特色籤（實證為本・創新導向・跨域合作・以人為本，文案可依實驗室現有定位微調）；右：大圖插槽（佔位漸層）
3. **研究核心領域**——四張圓角卡（現網站四主軸文案），圖示＋標題＋一句摘要
4. **最新消息**——試算表資料，卡片式 3 則預覽＋「查看全部」；分類籤（榮譽／發表／活動／公告）
5. **研究成果總覽**——統計卡 ×4，用真實數字：期刊論文 7・研討會論文 5・進行中計畫 4・實驗室成員 3（數字集中在 js/data.js 易更新；進場滾動）
6. **研究團隊**——成員卡預覽（主持人＋2 研究生，縮寫頭像佔位、職稱籤、專長一行）＋「查看全部成員」連 members.html
7. **聯絡我們＆加入我們**（`id="contact"`）——左：mailto 表單（姓名、Email、主題下拉、訊息）；右：三張引導卡（加入研究團隊／實習與見習／合作與交流）
8. **頁尾**——淺粉底：logo＋一句使命、聯絡資訊（賀彥中 助理教授／國防醫學大學護理學系／nokia3350g@gmail.com）、快速連結、版權列

## 6. Google 後台資料架構（沿用前版）

### 試算表（一份、兩工作表、發布到網路）
- **news**：`date`（YYYY-MM-DD）、`category`（榮譽｜發表｜活動｜公告）、`content`、`link`（選填）
- **events**：`date`、`title`、`description`、`image`（選填）、`pinned`

讀取：`https://docs.google.com/spreadsheets/d/{ID}/gviz/tq?tqx=out:json&sheet={name}`，依日期新→舊排序。

### 雲端資料夾（活動照片）
- Drive API v3 `files.list`（API 金鑰由使用者於 Google Cloud Console 建立、限制 referrer）
- 縮圖 `drive.google.com/thumbnail?id={id}&sz=w800`；檔名（去副檔名）＝照片說明
- 合作交流頁照片牆列全部；未設金鑰時顯示備用佔位照片

### 交付附「維護說明」
README 教老師：試算表加消息、上傳照片、檔名命名建議、素材規格清單用法、`firebase deploy`。

## 8. 測試與驗收

- 桌機（1280+）／平板（768）／手機（375）三尺寸逐頁檢查（Preview 工具實測）
- 外部資料源斷線測試：試算表/Drive 失敗 → 備用內容正常顯示
- `prefers-reduced-motion` 開啟時動畫停用
- Lighthouse：效能與無障礙 ≥ 90
- 內容比對現網站：四主軸、4 計畫、2 研究生、12 篇論文、聯絡資訊零遺漏
- 佔位頁（教學資源／開放資源／合作交流佔位區）呈現「建置中」樣式而非空白
